# VortexADO MCP — Distribution Guide

This guide explains how to distribute VortexADO MCP to team members.

---

## Primary Distribution: GitHub \+ curl Installer

The recommended distribution method uses a public GitHub repository with a one\-command installer.

### For Users (Installation)

Team members install with a single command:

```bash
curl -fsSL https://raw.githubusercontent.com/badgujar-kavita/ado-mcp-server/main/install.sh | bash
```

The installer will:

* Clone the repository to `~/.vortex-ado` (MCP runtime only — not user config)
* Install dependencies and build
* Register the MCP in Cursor (`~/.cursor/mcp.json`)

**After installation**, users open their project folder in Cursor and run `/vortex-ado/ado-connect`. The wizard writes connection details to `<workspace>/.vortex-ado/config.json` (per-workspace) and stores the PAT in the OS keychain. See [docs/conventions.md](conventions.md). Then restart the MCP server (Settings → MCP → refresh vortex-ado).

### For Developers (Updates)

After making changes to the MCP:

1. **Commit and push** to the GitHub repository:
   ```bash
   git add .
   git commit -m "Description of changes"
   git push
   ```

2. **Users update** by re\-running the same curl command:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/badgujar-kavita/ado-mcp-server/main/install.sh | bash
   ```

The installer automatically detects existing installations and pulls the latest changes while preserving user credentials.

## What's in the Repository

| File/Folder | Purpose |
|-------------|---------|
| `bin/bootstrap.mjs` | Entry point that launches the MCP server |
| `src/` | TypeScript source code |
| `dist/` | Compiled JavaScript (generated by build) |
| `package.json` | Dependencies and scripts |
| `install.sh` | One\-command installer |
| `uninstall.sh` | One\-command uninstaller |
| `docs/` | User and developer documentation |
| `conventions.config.json` | Bundled fallback (sanitized) — generic placeholders only. Per-tenant config is written by `/ado-connect` to `<workspace>/.vortex-ado/config.json`. |

---

## Uninstallation

Users can uninstall with:

```bash
curl -fsSL https://raw.githubusercontent.com/badgujar-kavita/ado-mcp-server/main/uninstall.sh | bash
```

This removes the `~/.vortex-ado` directory and the Cursor MCP registration. The uninstaller prompts whether to keep or delete the credentials file.

---

## What Users Need

| Requirement | Notes |
|-------------|-------|
| **Node.js v18+** | Installer checks this |
| **Cursor IDE** | Latest version |
| **ADO PAT** | Each user creates their own |
| **Internet** | For initial install and updates |

Credentials are stored in the OS keychain (`vortex-ado` service) and connection details in `<workspace>/.vortex-ado/config.json` per-workspace — never in the repository. Legacy `~/.vortex-ado/credentials.json` is still read as a transitional fallback.

---

## Security Notes

* **Never commit** credentials or PATs to the repository
* Each user creates their own ADO PAT with their account
* The repository contains no secrets; credentials are local to each machine
* Re\-installation preserves existing credentials
